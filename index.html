介绍Git 使用 SHA-1 算法计算数据的校验和，通过对文件的内容或目录的结构计算出一个 SHA-1哈希值，作为指纹字符串。该字串由 40个十六进制字符（0-9及 a-f）组成，看起来就像是：24b9da6552252987aa493b52f8696cd6d3b00373Git 的工作完全依赖于这类指纹字串，所以你会经常看到这样的哈希值。实际上，所有保存在 Git数据库中的东西都是用此哈希值来作索引的，而不是靠文件名。 Git 管理项目时，文件流转的三个工作区域： Git 的工作目录 暂存区域 以及本地仓库 基本的 Git 工作流程如下： 在工作目录中修改某些文件。 对修改后的文件进行快照，然后保存到暂存区域。 提交更新，将保存在暂存区域的文件快照永久转储到 Git 目录中。 工作常用分为俩种情况： 在工作目录中初始化新仓库$git init # 初始化;产生一个 .git 文件 $git remote add origin git地址 # 添加远程仓库;origin-远程仓库名字可自定义 从现有仓库克隆$ git clone git://github.com/s
